# M1 / M1+：prefix KV 固化 + suffix-only 去噪 + 端到端评测

M1 在 M0（单步前向跑通、与 openpi 数值对齐）基础上，把推理路径从「每步重算 prefix+suffix
联合注意力」优化为**「prefix 一次 prefill 固化 K/V + suffix-only 逐步去噪」**，并落地到
Chameleon 的 LIBERO 离线评测，与 TensorRT / PyTorch(openpi) 三路横向对比。

> 架构总体设计见 `docs/arch.md`；M0 落地见 `docs/M0.md`。本文件覆盖 M1 与 M1+ 的实现与结论。

## 1. M1：优化后的推理路径

| 阶段 | 导出函数 | 说明 |
|------|----------|------|
| 视觉 | `embed_image` | SigLIP 视觉塔 + projector，得到图像 patch embedding |
| 语言 | `embed_language` | 语言 token embedding |
| Prefill | `prefill` | expert-0(PaliGemma) 在 prefix 上跑一次，**固化全部层的 K/V** |
| 去噪 | `denoise_step_kv` | expert-1(action) 只在 suffix 上前向，attend 到固化的 prefix K/V + suffix 自身 |

对比 M0 的 `denoise_step`（每步都重算 prefix+suffix 的联合注意力），M1 把不随去噪步变化的
prefix K/V 提到循环外只算一次，循环内只跑 50 个 suffix token，计算量大幅下降。

### 关键设计

- **K/V 固化**：`prefill` 输出每层 expert-0 的 K/V（fp32 存储，attention logits 走 fp32），
  作为 `denoise_step_kv` 的输入常驻显存。
- **Host Euler 去噪环**：10 步 Euler 积分在宿主侧循环，逐步调用 `denoise_step_kv`；
  RoPE 的 cos/sin 表与时间编码在 host 用 numpy 预算（见 `sample.py::make_rope_np`）。
- **CUDA Graph**：单步 `denoise_step_kv` 用 CUDA Graph 捕获，降低 kernel launch 开销。
- **M1 = M0 数值等价**：`compare.py`（mode B, `--kv`）验证 M1 与 M0 路径 bit-identical。

## 2. M1+：分段编译、padding、bf16 修复、Chameleon 集成

在 M1 打通后，为了端到端接入 Chameleon 评测，补了以下能力：

### 2.1 分段编译（解决参数膨胀）

`export_tvm` 的 packed 模式会把**整个 `Pi0Model` 的参数**打进每个 engine（775 参数 / fp32
约 13.4GB），即使只导出 `embed_image` 也如此。改法：`Pi0Model.__init__` 增加 `include`
参数，按需只构建 `vision` / `embed` / `backbone` 子模块（见 `pi0_model.py::include_for`、
`compile.py`、`compile_quant.py`）。denoise engine 因此从 ~6.7GB 降到 ~4.8GB。

### 2.2 bf16 LayerNorm 限制

TVM 的 `topi.nn.layer_norm` 不支持 bfloat16。SigLIP 有 55 个 LayerNorm，故在 `siglip.py`
新增 `LayerNormF32`：内部用 float32 计算、输出再 cast 回输入 dtype，使 `embed_image` 可在
bf16 下编译。

### 2.3 prefix padding

`Pi05TrtPipeline` 会把 prefix 补齐到静态长度，而 M1 原假设固定有效 prefix。改法：
`gemma_dual.py` 的 `_sdpa` / attention 增加 `add_mask` 加性 mask 入参，`pi0_model.py` 的
`prefill` / `denoise_step_kv` 接收 `prefix_mask` 与 host 侧 `suffix_cos/suffix_sin`
（RoPE offset 按真实有效 token 数计算）。`compare_pad.py` 做自洽测试：带 padding + mask
与不带 padding 两条路径输出 bit-identical，证明 mask 正确屏蔽了 padding token。

### 2.4 Chameleon 集成（Python 3.11 / 3.12 隔离）

`tvm_ffi` 针对 Python 3.12 编译（依赖 `PyErr_GetRaisedException`），而 openpi 评测跑在
Python 3.11。采用**双进程架构**：3.11 客户端 spawn 一个 3.12 worker，用 framed pickle
协议（4 字节长度前缀 + pickle）over stdin/stdout 通信。新增运行时包 `pi05_tvm/`：

| 文件 | 作用 |
|------|------|
| `runtime/pi05_tvm/worker.py` | 3.12 worker：加载 mlc-vla engine + 权重，响应 `sample` 请求 |
| `runtime/pi05_tvm/engine.py` | 客户端 `TvmWorkerClient`（管理 worker 生命周期）+ VIT TRT engine 加载 |
| `runtime/pi05_tvm/pipeline.py` | `Pi05TvmPipeline`：复用 TRT 的 vit embedding + prefix padding，denoise 走 TVM worker |
| `evaluate/tvm_only_runner.py` | Chameleon policy runner（注册名 `tvm_only`） |
| `evaluate/pt_tvm_compare_runner.py` | PT vs TVM 双路对比（注册名 `pt_tvm_compare`） |
| `configs/pi05_libero_tvm_eval.yaml` | 单路 TVM vs GT 评测配置 |
| `configs/pi05_libero_tvm_compare.yaml` | PT vs TVM 双路对比 + WebUI |

vit 仍复用 TRT engine，仅 PaliGemma prefill + suffix-only denoise 换成 TVM。
默认走图内 `denoise_loop_kv`（`tvm_loop=true`），可选整段 CUDA Graph（`tvm_cuda_graph`）。

## 3. 端到端评测结果（LIBERO 离线，动作 vs ground-truth）

评测设置对齐三路：固定噪声 `seed=0`、`num_samples=10`、`stride=1`。指标为预测动作与
LeRobot 真值的逐样本误差均值。

| 后端 | mean_cosine | mean_max_abs | mean_mean_abs | 稳态速度/sample |
|------|-------------|--------------|---------------|-----------------|
| PyTorch (openpi, float 参考) | 0.9992 | 0.058 | 0.0094 | ~165ms |
| TensorRT | 0.9991 | 0.070 | 0.0098 | ~170ms |
| TVM bf16 | 0.9968 | 0.176 | 0.0362 | ~760ms |
| **TVM fp16** | **0.9988** | **0.079** | **0.0115** | **~480ms** |

> 配置文件：`configs/pi05_libero_eval.yaml`（PT）、`pi05_libero_trt_eval.yaml`（TRT）、
> `pi05_libero_tvm_eval.yaml`（TVM）。三者显存无法共存，故分三次跑后横向对比。

### 3.1 精度差距根因：bf16 累加精度

初版 TVM(bf16) 的 `max_abs=0.176` 明显高于 TRT(0.070)。对照实验把 TVM 切到 fp16 重跑，
误差立刻降到 **0.079**（逼近 TRT），cosine 从 0.9968 升到 0.9988。

结论：差距来自 **bf16 精度损失**，非逻辑 bug：

- bf16 尾数仅 7 位（fp16 有 10 位）；
- TVM/dlight 的 bf16 GEMM 累加精度低于 TensorRT 对 bf16 默认的 fp32 累加；
- 经 18 层 prefill + 10 步 Euler 去噪累积放大后，bf16 误差被显著放大；fp16 尾数更多，误差小得多。

fp16 顺带更快（760ms→480ms，此 GPU 上 dlight 的 fp16 kernel 比 bf16 更优）。因此
`pi05_libero_tvm_eval.yaml` 默认 `tvm_dtype: float16`。

- [x] 三路评测：TVM fp16 精度逼近 TRT（cosine 0.9988 / max_abs 0.079）。
- [x] 图内整段 Euler 环 `denoise_loop_kv` + 可选整段 CUDA Graph（`tvm_loop` / `tvm_cuda_graph`）。
- [x] PT vs TVM 双路对比 runner（`pt_tvm_compare`）+ WebUI 图例 PT−TVM。

### 剩余性能差距（下一步）

host Euler 路径下 fp16 约 ~480ms/sample（约 TRT 的 2.8×）。启用 `tvm_loop=true`
（默认）后，10 步 Euler 在图内一次执行，跨进程只传一次 prefix/noise/actions；再开
`tvm_cuda_graph=true` 可捕获整段去噪环。请用 `pi05_libero_tvm_eval.yaml` 重测稳态
latency，并与 TRT 对齐。

仍可继续优化：

1. vit 也迁到 TVM（去掉 TRT 依赖，便于非 NVIDIA 目标）；
2. 量化（int4/int8）与 dlight/MetaSchedule 调优；
3. 预编译 engine 落盘，避免每次 worker 启动时 JIT。
