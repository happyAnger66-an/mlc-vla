# M2：图内解码环 + cuBLAS/CUDA Graph + 真实权重 loader

M2 在 M0（GPT 主干 + KV-cache `prefill/decode_step`）、M1（视觉 tokenizer + 宿主 AR 环）
之上补齐三块，对齐 arch.md 路线图的 M2 目标（`dlight/cuBLAS + CUDA Graph + 图内 decode_loop_kv`
与真实权重对拍）：

1. **图内整段解码环 `decode_loop_kv`**：把宿主 18 步环下沉进计算图，一次调用跑完，配合 CUDA Graph 整环重放；
2. **编译 pipeline 升级**：CUDA 目标可选 cuBLAS BYOC（吃掉转置税）+ CUDA Graph，默认 pipeline 内 dlight 兜底；
3. **真实权重 loader**：把 plannn3 checkpoint 的 **GPT 主干 + 轨迹头** 映射到 relax nn 参数命名，为数值对拍打底。

> 总体方案见 `docs/p3/arch.md`；M0 见 `docs/p3/M0.md`，M1 见 `docs/p3/M1.md`。
> 照搬 mlc-vla（π0.5）姿势：`denoise_loop_kv` ↔ `decode_loop_kv`、`compile.py` ↔ `plannn3_compile.py`、
> `pi0_loader.py` ↔ `plannn3_loader.py`。

| 模块 | 文件 | 说明 |

| 图内解码环 | `model/plannn3/plannn3_model.py: decode_loop_kv` | prefill + 固定 `pred_times-1` 步 AR，逐步 RoPE/mask/onehot 编译期 bake，图内 argmax+embed |

## 2. 图内 `decode_loop_kv`

对齐 mlc-vla `denoise_loop_kv`：把宿主 `Plannn3Runner.generate` 的整个环搬进图。

```text
logits, kv = prefill(token_embeds)          # 复用 M0 prefill 方法体
id0 = argmax(logits)                          # 图内 topi.argmax
for step in range(pred_times-1):              # 固定步数，可整段 CUDA Graph
    pos = prompt_len + step                    # 完全确定 → cos/sin/add_mask/write_onehot 编译期 bake 为常量
    emb = embed_token(id_prev)                 # 图内 take 查表
    logits, kv = decode_step(emb, cos, sin, add_mask, write_onehot, kv)   # 复用 M0 decode_step 方法体
    id = argmax(logits)
return concat(ids) -> [1, pred_times]          # int32
```

要点：
- **直接复用 `prefill` / `decode_step`**：与宿主逐步环逐算子一致，数值天然等价（同算子、同 dtype）。
- **逐步量全 bake**：位置 `pos=prompt_len+step` 编译期已知，RoPE cos/sin、加性 mask、写入 onehot 都是常量，
  无需宿主每步传 tensor，消除 host↔device 往返 → 便于整段 CUDA Graph 捕获（一次 launch 重放全部 18 步）。
- **argmax 在图内**：`topi.argmax`（贪心），对齐 `resource/plannn3/infer.py` 的 `torch.argmax`，为 `traj_ids` bit-exact 对拍。

## 3. 编译 pipeline（cuBLAS / CUDA Graph / dlight）

`plannn3_compile._build_and_compile` 现支持（对齐 `mlc_vla.compile`）：

```python
# CUDA：matmul 卸载 cuBLAS + 吃掉转置税（cublas=None 自动探测，不可用回退 dlight）
mod = partition_for_cublas(mod); mod = RunCodegen()(mod); mod = FuseTransposeMatmul()(mod)
# CUDA Graph：decode 单步或 decode_loop_kv 整环捕获
with tgt, PassContext(config={"relax.backend.use_cuda_graph": True}):
    ex = relax.build(mod, target=tgt, relax_pipeline=get_default_pipeline(tgt))  # 默认 pipeline 含 dlight
```

- `cublas` 三态：`None`（默认，CUDA 且扩展可用则开）/`True`/`False`；不可用时 `resolve_cublas` 告警回退 dlight。
- `cuda_graph`：仅 CUDA 目标生效；`c`/`llvm` 忽略。
- `c` 目标仍走 zero pipeline + gcc（CPU 冒烟/对拍）。

## 4. 真实权重 loader

`plannn3_loader.py` 覆盖 **GPT 主干 + 轨迹头**（键名对照 `resource/plannn3/model/network.py`）：

| 源（torch state_dict） | 目标（relax nn） |
|---|---|
| `transfomer.h.{L}.{ln_1,ln_2}.weight` | `h.{L}.{ln_1,ln_2}.weight` |
| `transfomer.h.{L}.attn.{c_attn,c_proj}.weight` | `h.{L}.attn.{c_attn,c_proj}.weight` |
| `transfomer.h.{L}.mlp.{c_fc,c_proj}.weight` | `h.{L}.mlp.{c_fc,c_proj}.weight` |
| `traj_head.{ln,head}.weight` | `traj_head.{ln,head}.weight` |

- `load_state_dict(path)`：读 `.safetensors`/`.bin`（兼容 `{"state_dict":...}`/`{"model":...}` 包裹），bf16/fp16 上转 fp32，去 `module.` 前缀、`gamma→weight`。
- `load_params(config, src, named_params=...)`：按映射构造并逐参数 shape 断言；`embed.weight` 列入 `allow_missing`。
- `to_tvm_params(named_params, params, dev)`：按导出顺序打成 tvm ndarray（`allow_missing` 用 0 占位）。

**遗留项**：`embed.weight` 是 M0 对 `traj_encoder.ids_to_embed` 的替身，真实 checkpoint **无**对应单一权重
（离散 PCA tokenizer + learnable embedding）。真正的 traj token → embedding 需复刻
`trajectory_encoder_v2.TrajTokenizer.ids_to_embed`，属后续 milestone；因此当前主干+头可做**结构/主干对拍**，
端到端 `traj_ids` bit-exact 待 tokenizer 落地后闭环。

## 5. 如何运行

```bash
export TVM_HOME=/path/to/tvm
export TVM_LIBRARY_PATH=$TVM_HOME/build/lib
export PYTHONPATH=$TVM_HOME/python:$TVM_HOME/3rdparty/tvm-ffi/python:$(pwd)/python:$PYTHONPATH
# 或 source /path/to/Chamleon/scripts/tvm_env.sh
export TVM_FFI_DISABLE_TORCH_C_DLPACK=1   # 关键：跳过卡死的 torch dlpack JIT，import ~1.7s

# 图内整段解码环（dummy 秒级）
python -m mlc_vla.plannn3_compile --dummy --loop --target c

# CUDA：整环 + CUDA Graph（+ 自动 cuBLAS）
python -m mlc_vla.plannn3_compile --loop --target cuda --cuda-graph

# 打印 decode_loop_kv 的 IR
python -m mlc_vla.plannn3_compile --dummy --loop --dump-ir   # （经 build_irmodule；如需单函数可扩展）
```

程序化：

```python
from mlc_vla.model.plannn3 import Plannn3Config, load_state_dict, load_params, to_tvm_params
from mlc_vla.plannn3_runner import Plannn3Runner

cfg = Plannn3Config()
runner = Plannn3Runner(cfg, target="cuda", cuda_graph=True)          # 编 embed_token/prefill/decode_step/decode_loop_kv
src = load_state_dict("/path/to/pytorch_model.bin")                  # 真实 checkpoint
params = load_params(cfg, src, named_params=runner.named_params)     # 主干+头（embed.weight 缺省）
runner.set_params(to_tvm_params(runner.named_params, params, runner.dev))
ids = runner.generate_graph(token_embeds)   # 或宿主逐步：runner.generate(token_embeds)
```

## 6. M2 DoD

- [x] `--dummy --loop --target c`：`decode_loop_kv` legalize/build/run，产出 `[1,pred_times]` int32、图内 argmax 自洽。
      （已实测：`traj_ids=[7,31,4,30]`，tvm import 1.7s、build+run 秒级。）
- [x] `generate_graph` 与宿主 `generate` **同权重同输出**（`c` 目标实测一致：`[2,7,14,28]`）——图内环 ≡ 宿主环。
- [x] `build_name_map` 覆盖主干+头全部 14 个参数、shape 一致；仅 `embed.weight` 未映射（在 allow_missing）。
- [ ] `--loop --target cuda --cuda-graph`：CUDA Graph 整环捕获跑通（**需 GPU**；当前开发机 `cudaGetDeviceCount` Error 304，无 CUDA）。
- [ ] （对拍）主干 `prefill` 输出 vs `network.prefill`（同权重、同 token_embeds）连续量小容差对齐（需真实 checkpoint）。

> 运行注意：本机 `tvm_ffi` 首次导入会尝试 JIT torch dlpack 插件并卡死；**务必内联设置**
> `TVM_FFI_DISABLE_TORCH_C_DLPACK=1`（非致命、我们只用 numpy），import 即降到 ~1.7s。
