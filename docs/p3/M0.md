# M0：plannn3 GPT 主干 + KV-cache 在 TVM 跑通

M0 是 plannn3 落地开源 TVM 的第一步，目标：用 `tvm.relax.frontend.nn` 重写 plannn3 的
**GPT 主干 + KV-cache prefill/decode**，`target=c`/`llvm` 冒烟跑通，shape/dtype 自洽。

> 总体方案见 `docs/p3/arch.md`。本文件只覆盖 M0 落地细节与现状。
> 照搬 mlc-vla（π0.5）M0 的姿势：纯开源 TVM Relax、batch=1、固定 shape、宿主侧编排。

## 1. 已实现

| 模块 | 文件 | 说明 |
|------|------|------|
| 配置 | `model/plannn3/plannn3_config.py` | n_embd=1024/n_head=16/n_layer=12、vocab=1169、prompt/pred_times，含 `dummy()` 小尺寸 |
| 模型 | `model/plannn3/plannn3_model.py` | GPT 主干（interleaved RoPE、LayerNorm-no-bias、GELU-MLP）+ TrajHead + 三个可导出函数 |
| 编译/冒烟 | `plannn3_compile.py` | 导出 IRModule、最小 relax pipeline、随机权重跑 `prefill`+`decode_step` |

三个可导出函数（`get_default_spec`）：

- `embed_token(input_ids[1,1] int32) -> [1,1,n_embd]`：解码侧 traj token 的 embedding（`ids_to_embed` 的 M0 替身）。
- `prefill(token_embeds[1,prompt_len,n_embd]) -> (logits[1,1,vocab], kv[n_layer,1,max_seq,2*n_embd])`：跑一遍 prompt，返回首步 logits 与每层打包好的 KV（已 padding 到 `max_seq`）。
- `decode_step(latest_embed[1,1,n_embd], step_cos, step_sin, add_mask, write_onehot, kv) -> (logits[1,1,vocab], kv_new)`：单步自回归。

## 2. 关键设计取舍（M0 优先正确性）

- **不用 PagedKVCache**：KV 是预分配的 `[n_layer,1,max_seq,2*n_embd]` 缓冲，逐算子对齐
  `Plannn3VanillaRelay` 的 eager 路径；CUDA Graph / dlight 留到后续 milestone。
- **变长解码 → 固定 shape**：query 恒 `[1,1,n_embd]`；新 k/v 用 `write_onehot`（pos 处为 1）
  以「乘加」`past*(1-onehot)+new*onehot` 写入指定槽位，避免 `where`/动态 shape；
  有效长度、RoPE 位置、注意力 mask 全部由宿主按 `valid_kv_len` 算好后作为**运行时张量**传入，
  保证 `torch.jit.trace` 等价的 trace 稳定性（一张图跑完全部 18 步）。
- **interleaved RoPE**：偶/奇通道交错旋转，对齐 `network.apply_rotary_emb_interleaved`
  （区别于 π0.5/Gemma 的半分旋转）；cos/sin 恒 fp32，prefill 段 bake 为常量，decode 段宿主传入。
- **prefill causal mask / 末 token 选择器 bake 为常量**：末 token 用 one-hot `sel @ x` matmul
  选取（规避 `strided_slice` 的后端差异）。
- **dtype=float32**：便于 CPU 对拍；bf16/混精（视觉塔 fp32、LLM fp16）在后续 phase 切换。
- **重前端延后**：多相机视觉 tokenizer（DINOv3/EVA）、navi/history_traj 编码器**不在 M0**，
  M0 的 `token_embeds` 直接作为输入（宿主侧或后续 milestone 提供）。

## 3. 如何运行

前置：可用的 TVM（Unity/Relax，含 `tvm.relax.frontend.nn` 与对应运行库）。

```bash
export TVM_HOME=/path/to/tvm
export TVM_LIBRARY_PATH=$TVM_HOME/build
export PYTHONPATH=$TVM_HOME/python:$(pwd)/python:$PYTHONPATH

# 1) 打印导出的 relax IRModule（dummy 小尺寸）
python -m mlc_vla.plannn3_compile --dummy --dump-ir

# 2) 冒烟：随机权重跑通 prefill + decode_step
python -m mlc_vla.plannn3_compile --dummy --smoke --target c    # dummy 秒级
python -m mlc_vla.plannn3_compile --smoke --target c            # 全尺寸（c 后端编译较慢）
```

后端说明同 π0.5 M0：若本机 TVM 未链上 LLVM codegen，用 `--target c`（zero pipeline
合法化 → gcc 编译 `.so` → 加载运行）；链上 LLVM 或有 GPU 时直接切 `--target llvm/cuda`。
